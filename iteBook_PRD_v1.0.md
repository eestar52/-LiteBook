## 2. 项目背景

### 2.1 市场背景
随着移动互联网的普及，个人财务管理需求日益旺盛。根据艾瑞咨询数据，中国个人理财App市场规模预计2025年突破百亿元。然而，现有主流记账产品（随手记、挖财等）存在以下核心痛点：
1. **操作路径冗长**：用户需点击3-5步才能完成一次记账，使用门槛高。
2. **分析浮于表面**：数据分析停留在简单图表层面，缺乏消费行为洞察与个性化建议。
3. **UI设计老旧**：年轻用户（Z世代）审美偏好匹配度低。
4. **AI能力不足**：无法实现智能分类与消费预警。

### 2.2 商业机会
市场存在明显空白：面向18-35岁年轻用户、主打「轻量极速记账 + AI消费洞察」的差异化产品尚属稀缺。「轻记账 LiteBook」定位于此，以极简体验 + 智能分析为核心竞争力切入市场。

### 2.3 竞品分析

| 维度 | 随手记 | 挖财记账 | 轻记账 LiteBook（本产品） |
| :--- | :--- | :--- | :--- |
| **核心定位** | 功能全面的记账工具 | 记账+理财综合平台 | **极简记账 + AI消费洞察** |
| **目标用户** | 大众用户 | 理财投资人群 | **18-35岁年轻用户** |
| **记账速度** | 3-5步操作 | 4-6步操作 | **1步快速记账** |
| **AI能力** | 无 | 基础分类 | **智能分类 + 消费预警** |
| **UI风格** | 功能导向，偏复杂 | 金融风格，较严肃 | **极简清爽，年轻化** |
| **盈利模式** | 会员+广告 | 理财产品分佣 | **Premium会员** |

## 3. 目标用户分析

### 3.1 用户画像

| 属性 | 描述 |
| :--- | :--- |
| **年龄** | 18-35岁 |
| **职业** | 在校大学生、职场新人、白领 |
| **月收入** | 2,000 - 20,000元 |
| **核心诉求** | 了解自己的钱花在哪、控制冲动消费、养成良好消费习惯 |
| **使用场景** | 购物结账后、月末对账、日常闲暇浏览 |
| **痛点** | 记账繁琐容易放弃、看完报表不知如何改进、缺乏消费预警 |

### 3.2 用户故事（User Story）
* **大学生**：我希望能在3秒内完成一次记账，这样我就不会因为麻烦而放弃。
* **职场新人**：我希望App能帮我分析哪类消费占比最高，这样我能有针对性地节流。
* **注重储蓄者**：我希望在快超出预算时收到提醒，这样我能控制非必要消费。

### 3.3 用户旅程地图

| 阶段 | 用户行为 | 用户想法 | 痛点 | 机会点 |
| :--- | :--- | :--- | :--- | :--- |
| **发现** | 在App Store搜索记账 | 有没有好用又好看的记账App？ | 搜索结果同质化严重 | 突出「极简+AI」差异化定位 |
| **首次使用** | 注册、设置预算 | 希望快速上手 | 注册流程繁琐 | 简化注册，3步完成 |
| **日常记账** | 消费后打开App记录 | 希望越快越好 | 步骤多、分类乱 | 1步极速记账+AI自动分类 |
| **洞察分析** | 月末查看消费报告 | 想知道哪里花多了 | 图表看不懂、无建议 | AI生成个性化消费分析 |
| **养成习惯** | 持续记账打卡 | 希望坚持下去 | 缺乏激励机制 | 连续记账勋章与提醒 |

## 4. 产品目标与成功指标

### 4.1 产品目标
* **北极星指标**：日活跃用户数（DAU）
* **短期目标（上线3个月）**：获取1万DAU，次日留存率 > 40%
* **中期目标（上线1年）**：实现10万DAU，付费转化率 > 5%

### 4.2 成功指标（OKR）

| 目标（O） | 关键结果（KR） | 衡量方式 |
| :--- | :--- | :--- |
| **提升用户留存** | 次日留存率 > 40% <br> 7日留存率 > 25% | 数据埋点统计 |
| **提升记账效率** | 平均单次记账耗时 < 10秒 | 用户行为日志分析 |
| **提升价值感知** | 月活用户中查看分析报告比例 > 60% | 功能点击率统计 |
| **实现商业变现** | Premium会员付费率 > 5% | 支付系统数据 |

## 6. 功能详细需求 (核心模块)

### 6.1 极速记账
* **交互流程**：用户点击首页悬浮「+」按钮 -> 底部弹出记账面板 -> 输入金额 -> AI自动推荐分类 -> 点击保存，动效反馈后面板收起。
* **需求规则**：
  * AI推荐分类准确率目标 > 80%。
  * 保存成功需有震动+动效反馈，耗时 < 300ms响应。
  * 支持离线记账，联网后自动同步。

### 6.2 月度消费报告
* **功能描述**：首页展示当月消费总览，用户可进入报告详情页查看分类占比饼图、每日消费趋势折线图，以及Top3消费品类分析。
* **需求规则**：图表加载时间 < 1秒（本地数据渲染），支持按月切换，最多查看近12个月数据。

### 6.3 预算管理
* **功能描述**：用户配置月度总预算及各分类子预算。
* **需求规则**：达到预算80%时发送Push通知预警；达到100%时记账界面显示红色超支提示。
