## 💡 解决的核心问题

现有记账App的三大痛点：

1. **操作繁琐**：主流记账App平均需3-5步才能完成一次记账，用户易放弃
2. **分析肤浅**：数据仅停留在图表层面，缺乏个性化消费洞察与建议
3. **AI能力不足**：分类全靠手动，无法智能识别，无预算预警机制

---

## ✨ 核心功能（V1.0 MVP）

### 1. 极速记账
- 首页悬浮「+」按钮，一步唤起记账面板
- AI 根据历史行为自动匹配消费分类（准确率目标 >80%）
- 支持离线记账，联网后自动同步
- 单次记账目标耗时 **< 10秒**

### 2. 月度消费报告
- 首页实时展示本月消费汇总与预算进度
- 分类占比饼图 + 每日趋势折线图
- Top3 消费品类分析

### 3. 预算管理与预警
- 支持月度总预算 + 分类子预算配置
- 消费达 80% 时 Push 通知预警
- 超 100% 时记账界面显示超支提示（红色标注）

## 🏊 流程泳道图

> 文件：`design/swimlane.html` —— 用浏览器直接打开即可查看

覆盖两大核心业务场景：

**流程一：用户注册与首次设置**
`打开App → 手机号注册 → 验证码校验 → 设置月度预算 → 进入首页`

**流程二：日常快速记账（核心流程）**
`点击「+」→ 输入金额 → AI推荐分类 → 确认/修改 → 保存 → 预算阈值检查`

**参与角色（泳道）**：用户 / App前端 / AI服务 / 服务器

## ⚠️ 主要风险识别

| 风险 | 应对策略 |
|------|---------|
| 头部产品复制AI功能 | 加快迭代速度，建立用户使用习惯壁垒 |
| AI分类准确率不达标 | 人工标注训练集 + 用户纠错反馈机制 |
| 用户留存低于预期 | 前3个月密集运营，快速分析流失原因 |
| 用户数据安全 | AES-256加密 + HTTPS + 隐私合规审查 |
